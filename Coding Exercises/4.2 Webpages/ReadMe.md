# 4.2 Webpages

## Objective

Learn how to connect multiple webpages and resources within a website using relative links and folder structures.

## Description

This exercise demonstrates how webpages, images, and other resources can be organized into separate folders and connected using relative paths. The project includes a homepage, image assets, and additional webpages stored in different directories.

## Key Learning

* Create links between multiple HTML pages using anchor elements (`<a>`).
* Use relative paths to access files in different folders.
* Organize website files using a structured directory hierarchy.
* Link images and webpages from separate directories.
* Understand the relationship between web pages and project assets.

## Output

The website displays a homepage containing an image and a navigation link to another webpage. All resources are loaded correctly using relative file paths.

## Files

* `index.html` – Main homepage
* `solution.html` – Reference solution
* `assets/` – Contains website assets such as images
* `public/` – Contains additional webpages
* `goal.png` – Expected output
* `ReadMe.md` – Exercise documentation
